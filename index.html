    Pilot

    September 07, 2013 by Jason Lollback

    Choosing a publishing platform for a personal website hasn't been a trivial process. I've had to work though identifying systems that would meet some basic criteria that are important to me:

    • Low administration: Time should be spent on content and readability, not on infrastructure and databases.
    • Support for a responsive design. Mobile and tablet devices have quickly become the primary way that digital content is consumed and separate mobile sites aren't a good experience for the reader.
    • Speed: Poor hosting, slow networks or underpowered infrastructure can all make for a slow website.
    • Openness: I want to have control of the data and the technical platform.

    I've found that the best fit for these requirements is a workflow based on a static site generator, which is a relatively minor evolution of how simple web publishing began.

    The Static Website Generator Workflow

    1. Write content in plain text files using Markdown formatting.
    2. The static generator converts Markdown to HTML.
    3. Upload HTML files to a hosting site, which could be a hosting service, GitHub or Amazon S3.

    The static site generator takes my markdown formatted text files and converts them to HTML. Index and archive pages are automatically created and final HTML files are then uploaded to a webserver.

    There are many implementations of static site generators which means that if in the future I want to switch to a different engine I'm not locked in.

    Writing is an essential skill for any technology worker. Being able to manage a content publishing system complete with database infrastructure shouldn't be, which is why a static site generator is ideal for my needs.
